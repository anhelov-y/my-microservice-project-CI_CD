# Terraform (lesson-5)

## Структура проєкту

**`main.tf`** — головний файл конфігурації, що об'єднує всі модулі в єдину інфраструктуру.
**`variables.tf`** — опис глобальних змінних для гнучкого налаштування параметрів (регіон, назви ресурсів).
**`outputs.tf`** — визначення вихідних даних, таких як VPC ID та ECR URL, для подальшого використання.
**`backend.tf`** — конфігурація для зберігання файлу стану (State) у віддаленому S3 бакеті з підтримкою блокування через DynamoDB.
**`modules/`** — папка з перевикористовуваними модулями:
_ `vpc/` — мережева логіка.
_ `ecr/` — сховище контейнерів. \* `s3-backend/` — інфраструктура для самого Terraform.

## Опис модулів

### 1. Модуль VPC (`modules/vpc`)

Створює фундамент мережі в регіоні `us-west-2`.
**Ресурси**: VPC, Internet Gateway, публічні та приватні підмережі, NAT Gateway.
**Призначення**: Забезпечення ізоляції ресурсів та маршрутизації трафіку.

### 2. Модуль ECR (`modules/ecr`)

Налаштовує **Amazon Elastic Container Registry**.
**Ресурси**: Приватний Docker-репозиторій.
**Призначення**: Зберігання та версіонування образів застосунку для подальшого деплою.

### 3. Модуль S3 Backend (`modules/s3-backend`)

Створює ресурси для безпечного зберігання стану проєкту.
**Ресурси**: S3 Bucket з шифруванням та DynamoDB Table для Lock-файлів.
**Призначення**: Запобігання конфліктам при одночасній роботі кількох розробників (State Locking).

## Команди для керування інфраструктурою

Використовуйте ці команди в терміналі вашої віртуальної машини (Linux):

## 🛠 Основні команди

- `terraform init` — ініціалізація проекту.
- `terraform plan` — перегляд змін перед створенням.
- `terraform apply` — створення інфраструктури в AWS.
- `terraform destroy` — повне видалення всіх ресурсів (див."Усунення несправностей").

## Робота з Backend та Lock

- `terraform init -migrate-state` — переніс стейту (наприклад, з локального в S3).
- `terraform force-unlock <LOCK_ID>` — розблокування стейту при помилках.

## Усунення несправностей

- Якщо Terraform "завис" на Lock: перевірте наявність таблиці DynamoDB.
- Якщо `prevent_destroy = true`: змініть на `false` у коді модуля перед видаленням.
- Для повного скидання: видаліть папку `.terraform` та файл `.terraform.lock.hcl`.

## Очищення ресурсів

Завжди виконуйте `terraform destroy` після завершення роботи, якщо проект був створений в навчальних цілях або ви до нього
більше не збираєтесь повертатись у маймутньому (супроводжується перевіркою даних в AWS для точного видалення).
